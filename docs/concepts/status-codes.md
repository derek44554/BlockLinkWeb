# 状态码

BlockLink 使用统一的状态码规范来描述请求与操作的结果，状态码按类别分组，十位数字代表类别。

## 2X — 成功

| 状态码 | 说明 |
|--------|------|
| 21 | 正确处理 |

## 3X — Block 获取错误

| 状态码 | 说明 |
|--------|------|
| 31 | BID 不存在 |
| 32 | 请求错误，不予处理（一般为接收的数据存在问题） |
| 33 | 请求错误，不予处理（一般为服务器问题） |

## 4X — 请求错误

| 状态码 | 说明 |
|--------|------|
| 41 | 路由不存在 |
| 42 | 请求体错误 |

## 5X — 连接相关

| 状态码 | 说明 |
|--------|------|
| 51 | 未连接 / 未被注册 |
| 52 | 拒绝注册 |
| 53 | 已连接，并正在与新节点进行连接 |

## 6X — 转发相关

| 状态码 | 说明 |
|--------|------|
| 61 | 指令无法转发到目标 BID，没有与该节点的连接路径（如 A → B → C 路径不通） |
| 62 | 指令无法由本节点发出，可能未与其他节点直接连接，或直连节点无法转发 |
| 63 | 本节点不负责转发此指令 |

## 7X — 权限相关

| 状态码 | 说明 |
|--------|------|
| 71 | 权限被拒绝（路由权限 / 节点权限 / Block 权限），绝对性拒绝 |
| 72 | 权限被拒绝，但非绝对，满足特定条件（如时间、请求内容等）后可能被接受 |
| 73 | 需要授权，除超级权限外，其他权限访问须经授权 |
