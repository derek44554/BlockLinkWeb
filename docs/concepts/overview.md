# BlockLink 概览

BlockLink = **Block**（数据块）+ **Node**（节点）

一套让你完全掌控自己数据的分布式生态系统。创建属于自己的私有节点网络，存储一切信息，让数据真正为你所用。

## 三层架构

### 数据层 — Block

块是 BlockLink 中数据存储的最小单元，可以存储任何信息——照片、视频、笔记、文章、联系人、账号、想法等，一切皆可成为块。

每个块都有：
- 唯一 BID（全网可寻址）
- 类型与标签
- 权限等级
- 与其他块的关联关系

块与块之间可以相互连接，通过标签组织检索，比传统文件夹分类强大得多。

### 网络层 — Node

运行 BlockLink 服务的计算机即为节点。节点负责提供功能，例如块的存储与读取，也可以扩展出任意自定义功能。

BlockLink 网络的通信方式与传统不同：

- **只需 BID，无需 IP** — 框架自动在全网定位目标节点或块，自动计算最优路径
- **跨 NAT 通信** — 通过中继转发，内网设备也能互相通信
- **局域网自动发现** — 同一网络内自动发现并直连
- 没有公网 IP 的设备不再是数据孤岛

### 扩展层 — 应用生态

节点是高度可扩展的，功能模块松耦合、可插拔，可随意接入或移除：

- 使用官方或社区扩展，快速为节点增加新能力
- 自行开发扩展，打造专属功能
- 基于 BlockLink 构建独立 App，数据在网络内互通共享

## 安全性

BlockLink 的安全体系基于**顶级密钥**：

- 只有经过顶级密钥签名的节点才能加入你的网络
- 签名过程基于密码学，完全离线也可完成
- 谁拥有顶级密钥，谁就拥有这个网络的所有权
- 每个节点和块都有独立的权限等级，需授权或足够权限才可访问
- 支持加密块，只有持有解密密钥的人才能查看内容，顶级密钥也无法解密

## 典型应用场景

- **个人知识库** — 将笔记、日记、文章存储为块，在所有设备间同步
- **私有相册 / 音乐库** — 数据存在自己的节点，不依赖任何第三方平台
- **分布式文件系统** — 构建自己的云存储，数据分散在多个设备
- **密码管理器** — 将账号信息存储为加密块，安全访问
- **自定义 App** — 基于 BlockLink 开发博客、内容管理、自动化工作流等
